Overview

I was lucky enough to intern at NASA's Glenn Research Center for the summer of 2023 as a test engineering intern. I worked at Glenn's Power Systems Facility where novel power systems for space applications are developed. I contributed to a project known as AMPS (Advanced Modular Power System) by creating a GUI to display system data.

Motivation

The AMPS project had a previous solution for the system data GUI. A display with an embedded microcontroller was being used, however, these displays proved to be unreliable. A failed display is pictured below (left).



The AMPS team wanted a new browser based GUI running on a Raspberry Pi. I was assigned the task of designing one from scratch.

Design

Major challenges of this project included:

- Communicating with DAQ units and reading valid sensor values
- Parsing DAQ readings and interpreting if they are within normal operating ranges
- Displaying values on a GUI which can be easily interpreted by a user

The final design is depicted above and below. A Raspberry Pi was used to communicate with two Omega D6400 DAQ units. An intermediary device was used to convert from USB to the RS485 protocol in order to read and write from the DAQs.
A timer relay was also used to function as a watchdog timer which would shut off if the system froze, or if the system detected a sensor value out outside of the normal operating range.

The Raspberry Pi was connected to and powered by a touchscreen display.

Javascript and Python were used for reading, parsing and displaying data. Below is a simple diagram showing how the MQTT protocol was used to interface Python with Javascript.

Results

The project was successfully completed and fully integrated into the AMPS rack.

In instances when sensor values stray beyond user defined limits, the system will respond by placing itself into an alarm state. In this alarm state, the timer relay controlled by the Raspberry Pi is opened and the system will shut down.

The system will remain in this state until the sensor value returns to normal and a user clears the alarm state. This behavior is demonstrated below.
